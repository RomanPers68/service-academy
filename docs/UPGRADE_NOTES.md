Всё ниже уже работает «из коробки» после деплоя, кроме двух необязательных
серверных шагов (помечены ⚙️). Приложение полностью функционально и без них.

## Этап 1 — включено сразу
- **Экзамен и сертификаты**: флаг `CERTIFICATES_ENABLED = true` в `App.jsx`.
- **Глобальный поиск** (`ui/search.jsx`): кнопка 🔍 на главном экране роли.
  Ищет по урокам текущей роли и глоссарию.
- **Оценка времени**: «≈ N мин» на карточках модулей, «осталось ≈ …» в прогрессе.
- **Интервальное повторение** («Работа над ошибками»): верный ответ двигает
  вопрос по интервалам 1 → 3 → 7 → 30 дней; 4 верных подряд — вопрос закреплён;
  ошибка сбрасывает прогресс. Старые сохранённые ошибки совместимы.

## Этап 3 — ⚙️ напоминания в Telegram («Вопрос дня»)
Приложение уже сохраняет Telegram ID пользователя (localStorage `sa_tg_id`)
и пытается отправить его на сервер через `rpcSync("sa_set_tg", …)`.
Чтобы рассылка заработала:

1. **Бот**: в @BotFather создай бота (или используй того, в котором открывается
   WebApp) и сохрани токен.
2. **Колонка и RPC** — Supabase → SQL Editor:
   ```sql
   alter table profiles add column if not exists tg_chat_id bigint;

   -- Привязка chat id к профилю по токену сессии.
   -- ВАЖНО: подставь сюда свою реальную логику поиска профиля по токену —
   -- посмотри, как это сделано в твоих существующих RPC-функциях записи.
   create or replace function sa_set_tg(p_token text, p_tg_id bigint)
   returns json language plpgsql security definer as $$
   begin
     update profiles set tg_chat_id = p_tg_id
     where session_token = p_token;  -- ← замени на свою связку токен→профиль
     return json_build_object('ok', true);
   end; $$;
   ```
3. **Функция**: `supabase functions deploy daily-reminder`
   (код: `supabase/functions/daily-reminder/index.ts`).
4. **Секреты**: Dashboard → Edge Functions → Secrets:
   `TG_BOT_TOKEN` = токен бота, `APP_URL` = ссылка на приложение.
5. **Расписание**: Dashboard → Edge Functions → daily-reminder → Schedules —
   cron, например `0 8 * * *` (проверь актуальный интерфейс в документации
   Supabase, разделы про scheduled functions менялись).

После этого каждый сотрудник, хоть раз открывший приложение в Telegram,
будет получать «Вопрос дня» перед сменой.

1. `npm install && npm run dev`
2. Экран ролей: появились плитки «Меню», «Допуск», «Сертификаты».
3. Главный экран роли: кнопка 🔍, время «≈ N мин» на модулях.
4. Открой урок: режим карточек, свайп, переключатель ☰ в шапке.
5. Завали вопрос в тесте → «Работа над ошибками» → ответь верно →

## Этап 4 — «Новинки»: PDF → команда учит → менеджер видит
Полный цикл: менеджер закидывает PDF с новыми позициями → AI разбирает его в
карточки блюд → команда видит плашку «Новые позиции · NEW» и учит →
менеджер видит «Кто выучил новинки».

Что уже работает без настройки:
- Кнопка «⚡ Импорт из PDF» и «Опубликовать команде» в редакторе меню.
- Плашка «Новые позиции» в тренажёре (блюда с пометкой isNew, 30 дней).
- «Выучил новинки ✓» в конце колоды — локальная отметка сразу.

⚙️ Два серверных шага (по 5 минут):
1. **AI-импорт**: Vercel → Settings → Environment Variables →
   `ANTHROPIC_API_KEY` (ключ: https://console.anthropic.com). Функция
   `api/menu-import.js` задеплоится сама вместе с проектом.
   Актуальные детали API: https://docs.claude.com/en/api/overview
2. **Общее меню и трекинг**: выполни `supabase-stage4.sql` в Supabase →
   SQL Editor (подставь свою связку токен→сотрудник, помечено в файле).
   После этого «Опубликовать команде» реально раздаёт меню всем устройствам,
   а «Кто выучил новинки» показывает картину по сотрудникам.

### Про размер и «гигабайты»
- Фото «Два моря, Океан» лежат в деплое: ~800 КБ на 10 блюд, грузятся лениво
  (loading="lazy") — на скорость запуска не влияют.
- Фото, добавленные менеджером с телефона, сжимаются до ~60–120 КБ и хранятся
  локально (localStorage, лимит ~5 МБ ≈ 30–50 блюд с фото на устройство).
- При публикации команде меню (JSON) уходит в таблицу Supabase — текст лёгкий.
  Если блюд с фото станет много, следующий шаг — Supabase Storage: фото по URL,
  1 ГБ бесплатного тарифа ≈ 12 000 фото. Гигабайты приложению не грозят.

## Этап 6 — «Книга отзывов» 📖
Фишка мотивации: каждый пройденный модуль превращается в отзыв благодарного
гостя в личной книге сотрудника. Навигация приложения НЕ менялась.

Новые файлы:
- `data/reviews.js` — 43 отзыва (под каждый модуль всех 5 ролей, добрые
  вперемешку со смешными), 5 легендарных страниц (за экзамены ролей, с
  сургучной печатью), «гость недели», звания (Стажёр → Официант зала →
  Мастер сервиса → Метрдотель → Амбассадор), хелперы bookStats/moduleDone.
- `ui/guestbook.jsx` — экран книги (бумажные страницы, рукописный шрифт
  Marck Script, печати, разделы по ролям, листание) + баннер NewPageBanner.

Точки интеграции (всё остальное не тронуто):
1. Плитка «Книга отзывов» — первой в сетке плиток на главной (RoleSelect).
2. Экран модуля: если модуль полностью пройден — золотой баннер «Гость
   оставил тебе новую страницу» (показывается один раз, sa_book_seen).
3. Профиль: звание и счётчик страниц под именем (bookStats).
4. «Гость недели»: страница-испытание в книге, открывает живой диалог
   (пул: last, anticipation, checkback1, yolochka, stoplist1; ротация по
   ISO-неделе). Завершение пишется в completed как weekly_<неделя>.
5. Экзамен роли сдан (examResults.passed) → легендарная страница с печатью.

Принципы: в книге только победы — провал не оставляет плохих страниц
(«гость ушёл без отзыва, но вернётся»). Страницы появляются задним числом
за уже пройденные модули. Даты страниц фиксируются при первом открытии
книги (localStorage sa_book_dates).

## Этап 7 — Главная в фирменном стиле «оправа + жетоны + книга»
Утверждённый по макетам редизайн главной (навигация не менялась):
1. Профильная карточка убрана — вместо неё живое приветствие по часам
   («Доброе утро/день/вечер/ночи, Имя» + ресторан); карандаш редактирования
   профиля остался рядом с именем. Полный профиль — во вкладке «Профиль».
2. Карточка «Твой трек» — первый непройденный урок роли с оценкой минут,
   кнопкой ДАЛЬШЕ и полосой прогресса. Цепочка при полном прохождении:
   уроки → вопросы на повторении (mistakeBank) → «Гость недели» (книга).
3. «Книга отзывов» — слим-витрина в полной золотой оправе: монограмма
   владельца, «ЛИЧНАЯ · Имя · Звание», сургучные печати (заработанные/лунки),
   золотая нить прогресса по нижней кромке, красное ляссе. Плитка книги из
   сетки инструментов убрана (заменена витриной).
4. Инструменты — «жетоны»: карточки в золотой оправе (двойной background
   padding-box/border-box) с латунным люверсом сверху; неполный ряд
   центрируется. Бейдж на «Меню» — реальное число новинок (countNewDishes,
   переехал в data/reviews.js и экспортируется).
5. Заголовок «Выбери свою роль» → «Треки обучения», когда роль уже выбрана.
Иерархия оправ: книга — полная (яркая), трек и жетоны — средняя,
карточки ролей/модулей — как в стилях (S/A уже содержат оправы).
Обе темы (тёмная и «Для чтения») поддержаны во всех новых элементах.
Изменены: ui/screens.jsx, App.jsx (пропсы RoleSelect), data/reviews.js,
ui/guestbook.jsx (countNewDishes → импорт).
