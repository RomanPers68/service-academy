# Подключение AI-ассистента «Наставник» (бесплатно)

Ассистент работает через Supabase Edge Function `ai-chat` — она хранит ключ
ИИ-провайдера на сервере и отвечает только сотрудникам с валидным кодом
(проверка через твою же функцию `whoami`). Провайдер — OpenRouter: один ключ,
десятки моделей, есть бесплатные. Настройка ~7 минут, по шагам:

## Шаг 1. Ключ OpenRouter (2 мин)
1. Зарегистрируйся на https://openrouter.ai (можно через Google/GitHub).
2. Меню → **Keys** → **Create Key** → назови «Service Academy» → скопируй ключ
   (начинается с `sk-or-…`). Больше он не покажется — сохрани.
3. Платить не нужно: используем модель с пометкой `:free`.

## Шаг 2. Развернуть функцию (3 мин)
1. Supabase → твой проект → в левом меню **Edge Functions**.
2. **Deploy a new function** → способ **Via Editor** (прямо в браузере).
3. Имя функции: `ai-chat` (ровно так).
4. Сотри заготовку и вставь целиком содержимое файла
   `supabase/functions/ai-chat/index.ts` из проекта.
5. **Deploy**.

## Шаг 3. Секреты (2 мин)
1. Edge Functions → **Secrets** (или Settings → Edge Functions → Secrets).
2. Добавь секрет: имя `OPENROUTER_API_KEY`, значение — ключ из шага 1.
3. (Необязательно) секрет `OPENROUTER_MODEL`, если хочешь другую модель.
   По умолчанию: `deepseek/deepseek-chat-v3-0324:free` — хороший русский,
   бесплатно. Альтернативы смотри на openrouter.ai/models (фильтр Free).

## Шаг 4. Проверка
Открой приложение → на главной жетон **«Ассистент»** → задай вопрос
(например, «Гость недоволен блюдом — что говорить?»). Ответ должен прийти
за 2–8 секунд.

## Если что-то не так
- **«Ассистент ещё не подключён»** — функция не задеплоена или нет секрета
  `OPENROUTER_API_KEY` (проверь имя — ровно так, без пробелов).
- **«Бесплатный лимит исчерпан»** — у free-моделей дневная квота
  (~50 запросов/день без пополнения; положи $10 на счёт OpenRouter — лимит
  free-моделей вырастет до 1000/день, деньги не списываются).
- **Долго думает / ошибка провайдера** — free-модели иногда перегружены:
  подожди минуту или смени модель через секрет `OPENROUTER_MODEL`.
- Логи функции: Edge Functions → ai-chat → **Logs** — там видно каждую ошибку.

## Когда пойдёте в прод
Смени секрет `OPENROUTER_MODEL` на платную модель (например,
`anthropic/claude-3.5-haiku` — быстрый и дешёвый, ~$1 за ~500 диалогов)
и пополни баланс OpenRouter. Код менять не нужно — только секрет.

# AI HR: функция `ai-hr` (интервьюер собеседований)

Вторая Edge Function — ИИ-интервьюер для «Собеседования кандидата».
Разворачивается так же, как ai-chat, за 3 минуты:

1. Supabase → **Edge Functions** → **Deploy a new function** → **Via Editor**.
2. Имя: ровно `ai-hr`.
3. Вставь целиком содержимое `supabase/functions/ai-hr/index.ts` → **Deploy**.
4. Секреты добавлять НЕ нужно: `OPENROUTER_API_KEY` общий для всего проекта
   и уже задан при настройке ai-chat.

Проверка: Команда → Собеседование кандидата → заполни имя и опыт →
на экране «Передайте телефон кандидату» появится кнопка
**«🤖 AI-интервью · бета»** — ИИ поздоровается и задаст первый вопрос.
После 4+ ответов появится «Завершить интервью → оценка»: результат
запишется в ту же историю собеседований, с вердиктом ИИ и оценками
по компетенциям (и уйдёт в облако, как обычные).

Важно: ИИ может ошибаться — финальное решение о найме всегда за менеджером
(эта приписка есть и на экране интервью).
